## 📚 完整框架缩写解释

### 1️⃣ 简单框架（3要素）

| 缩写 | 全称 | 中文含义 | 适用场景 | 一句话总结 |
|------|------|---------|---------|-----------|
| **ERA** | **E**xpectation, **R**ole, **A**ction | 期望、角色、行动 | 快速任务、日常AI交互 | 最基础的提示词模板 |
| **TAG** | **T**ask, **A**ction, **G**oal | 任务、行动、目标 | 简单内容生成、快速原型 | 极简三步法 |
| **APE** | **A**ction, **P**urpose, **E**xpectation | 行动、目的、期望 | 提示词优化、任务指令 | 强调"为什么"的简单框架 |
| **RTF** | **R**equest, **T**ask, **F**ormat | 请求、任务、格式 | 数据检索、操作指南 | 重点是输出格式控制 |
| **BAB** | **B**efore, **A**fter, **B**ridge | 之前、之后、桥梁 | 营销文案、产品推广 | 讲故事框架（问题→愿景→解决方案） |
| **PEE** | **P**oint, **E**vidence, **E**xplanation | 观点、证据、解释 | 学术写作、论文段落 | 学术论证三段式 |
| **ELI5** | **E**xplain **L**ike **I**'m **5** | 像给5岁孩子解释 | 复杂概念简化、科普 | 用最简单的语言解释 |

|------|------|---------|---------|-----------|
| **RACE** | **R**ole, **A**ction, **C**ontext, **E**xpectation | 角色、行动、上下文、期望 | 角色扮演、客服脚本 | ERA升级版，加了背景 |

## 💡 实际使用对比

让我用同一个任务展示不同复杂度的区别：

**任务：帮我写一篇关于Python数据分析的文章**

### 用ERA（简单）

```
Expectation: 1000字技术文章
Role: 你是Python数据科学家
Action: 写一篇关于Python数据分析的文章
```

**特点**：快速构建，适合简单任务

---

### 用RACE（中等）

```
Role: 你是有10年经验的数据科学家
Action: 写一篇Python数据分析入门文章
Context: 目标读者是刚学Python的大学生，有编程基础但没有数据分析经验
Expectation: 1500字，包含3个实际案例，代码示例用Pandas和Matplotlib
```

**特点**：增加了背景信息，输出更精准

---

### 用CRISPE（复杂）

```
Clarity: 写一篇面向初学者的Python数据分析教程
Relevance: 帮助读者快速掌握数据分析核心技能，为求职做准备
Iteration: 先写大纲，然后逐步完善每个章节
Specificity: 使用真实数据集（如泰坦尼克数据），包含数据清洗、可视化、基础统计分析
Parameters: 1500-2000字，至少3个代码示例，适合30分钟阅读
Examples: 参考《Python for Data Analysis》风格，简洁实用
```

**特点**：信息全面，质量最高，但需要更多思考时间

## ✅ 核心总结

### "要素"就是框架的组成部分数量

- **3要素** = 最基础，适合日常快速使用（ERA、TAG、APE等）
- **4-5要素** = 标准版，适合大部分工作任务（RACE、SMART、SPEAR等）
- **6+要素** = 专业版，适合复杂重要任务（RACEF、CRISPE、SCAMPER等）

### 形象类比：汉堡套餐

- **简单汉堡** = 面包+肉+面包（3层）→ 快餐店，快速解决温饱
- **标准汉堡** = 面包+肉+芝士+生菜+面包（5层）→ 普通餐厅，营养均衡
- **巨无霸汉堡** = 面包+肉+芝士+生菜+酱料+洋葱+面包（7+层）→ 高档餐厅，精心烹饪

**选择建议**：根据你的"饥饿程度"（任务复杂度）选择合适的框架！

## 🔍 记忆技巧

### 简单框架（都是3个字母）
- **ERA** = 时代（想象新时代的提示词）
- **TAG** = 标签（给任务打标签）

- **SMART** = 聪明（聪明的目标设定）
- **SPEAR** = 长矛（像长矛一样直击要害）

### 复杂框架（6+个字母）
- **SCAMPER** = 奔跑（创新思维像奔跑一样快）
- **CRISPE** = 酥脆（清晰明确像脆饼）

## 📖 使用建议

### 初学者路线
1. **第一周**：只用ERA（最简单）
2. **第二周**：尝试RACE（加入上下文）
3. **第三周**：根据场景选择专用框架（BAB、PEE、SMART等）
4. **第四周+**：掌握复杂框架（RACEF、CRISPE等）

### 专业用户建议
1. **日常任务**：ERA、TAG（快速高效）
2. **重要任务**：RACE、SMART（平衡质量和速度）
3. **关键项目**：RACEF、CRISPE（追求最佳效果）

### 场景映射表

| 工作场景 | 推荐框架组合 |
|---------|-------------|
| **产品经理** | SMART（目标）+ FOCUS（产品分析）+ SCAMPER（创新） |
| **营销人员** | BAB（文案）+ SPEAR（提案）+ CRISPE（策略） |
| **数据分析师** | RACE（报告）+ FOCUS（对比）+ CIDI（项目） |
| **技术写作** | PEE（论证）+ ELI5（简化）+ RTF（格式化） |
| **项目管理** | SMART（目标）+ CIDI（执行）+ RACEF（战略） |
