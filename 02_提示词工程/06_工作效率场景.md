# 模块6：工作效率场景

> 难度：⭐⭐⭐⭐ 高级 | 建议时长：4-5小时
>
> 目标：掌握写作、分析、办公、学习等高频工作场景的专业提示词技巧

---

## 6.1 写作场景

### 场景A：商业邮件

```
请写一封[邮件类型]，详细信息如下：

发件人身份：[你的职位]
收件人：[对方的职位和关系]
邮件目的：[你想达到什么效果]
关键信息：
- [要点1]
- [要点2]
- [要点3]
语气：[正式/半正式/友好]
字数：[xxx字以内]

特别注意：
- [任何特殊要求，如：避免催促语气]

输出：
1. 邮件标题（3个候选）
2. 邮件正文
```

**实际例子：**
```
请写一封催促邮件。

发件人：项目经理
收件人：合作方的技术负责人（合作2年，关系不错）
目的：催促对方按时交付API接口文档，已延迟1周
关键信息：
- 原定上周五交付，目前已延迟1周
- 我方下游开发团队在等这份文档
- 如果本周三前拿不到，整体上线时间会推迟
语气：友好但明确传达紧迫性
字数：200字以内

特别注意：不要让对方觉得被指责，重点是"我们一起解决"
```

### 场景B：公众号/自媒体文章

```
请为[平台]创作一篇关于[主题]的内容。

平台：[微信公众号/知乎/小红书/B站]
目标读者：[读者画像]
内容目的：[教育/种草/科普/观点输出]

内容要求：
- 标题：吸引人但不标题党，给3个候选
- 开头：前100字必须抓住注意力
- 结构：[总分总/故事线/问答式]
- 字数：[xxx字]

风格参考：
"[粘贴一段你喜欢的风格样本]"
```

### 场景C：工作文档

```
请撰写一份[文档类型]。

类型：[技术方案/需求文档/竞品分析/周报]
读者：[谁会看]
目的：[解决什么问题]

内容大纲：
1. [章节1]
2. [章节2]
3. [章节3]

素材：
"""
[你的笔记/数据/要点]
"""

格式：Markdown，关键数据用表格，结论用加粗
质量要求：有数据支撑，结论可执行，避免空话
```

## 6.2 数据分析场景

### 场景A：数据探索与可视化

```
我有一份 CSV 格式的销售数据，包含字段：
日期、产品名、销售量、单价、地区、客户类型

请用 Python（pandas + matplotlib）写分析脚本，完成：

1. 数据概览：行数、列数、缺失值、描述性统计
2. 销售趋势：按月汇总趋势图 + 环比增长率
3. 产品分析：各产品销售额占比饼图 + Top5柱状图
4. 地区分析：各地区对比 + 地区×产品热力图

要求：
- 图表用中文标题和标签
- 每步有注释说明
- 最后输出一段关键发现的文字总结
```

### 场景B：数据报告解读

```
以下是公司上月的运营数据：
"""
[粘贴数据或表格]
"""

请分析：
1. 关键指标：哪些好/差？与行业平均相比如何？
2. 趋势分析：与上月/去年同期对比，哪些趋势值得关注？
3. 异常诊断：数据中有没有异常值？可能原因？
4. 行动建议：基于数据建议哪些行动？按优先级排序

输出格式：先给1页管理层摘要（重点+结论+建议），
然后是详细分析。
```

### 场景C：SQL 查询

```
数据库表结构：
users（用户表）：id, name, email, created_at, vip_level
orders（订单表）：id, user_id, product_id, amount, status, created_at
products（产品表）：id, name, category, price

请写 SQL 查询：[描述你的查询需求]

要求：
- 标准 SQL 语法（兼容 MySQL 8.0）
- 关键步骤加注释
- 考虑大表性能，建议需要的索引
```

## 6.3 办公效率场景

### 场景A：会议纪要

```
以下是会议录音转文字稿：
"""
[粘贴文字稿]
"""

请整理为正式的会议纪要：

1. 会议基本信息（时间、参会人）
2. 议题与讨论要点（按议题分类）
3. 决议事项（清晰列出每项决议）
4. 待办事项（表格：负责人 | 任务 | 截止时间）
5. 下次会议安排

格式：正式公文风格，总字数控制在原文的 20%-30%
```

### 场景B：邮件批处理

```
以下是我收件箱中的 5 封邮件摘要：
"""
[粘贴摘要]
"""

请帮我：
1. 按紧急程度排序（紧急/重要/普通/可延后）
2. 每封邮件拟一个回复要点
3. 对需要立即回复的，草拟完整回复
4. 对可以批处理的，建议统一处理方式
```

### 场景C：知识管理

```
以下是我阅读[文章/书籍]的笔记：
"""
[粘贴笔记]
"""

请帮我：
1. 提炼核心观点（不超过5个）
2. 生成记忆卡片（问题→答案格式，每张一个知识点）
3. 建议 3 个可以实际应用这些知识的场景
```

## 6.4 学习教育场景

### 场景A：概念学习（费曼学习法）

```
请用"费曼学习法"帮我理解[概念名]。

1. 简单解释：用10岁小孩能听懂的方式
2. 生活类比：用一个日常事物来比喻
3. 核心要点：3-5 个关键知识点
4. 常见误区：最容易犯的 2-3 个错误
5. 检验理解：给我 3 个问题测试我是否真懂了

我的背景：[你的知识水平]
```

### 场景B：学习路径规划

```
我想学习[技能/领域]。

现状：
- 当前水平：[零基础/有基础/中等]
- 每周可用时间：[x小时]
- 学习目标：[达到什么水平]
- 期限：[x个月]

请制定：
1. 学习路线图（基础→进阶的路径）
2. 分阶段计划（每阶段目标和内容）
3. 推荐资源（书/课/网站，标注难度和优先级）
4. 每周安排（表格）
5. 里程碑检查点（怎么判断学会了）
6. 常见的坑（过来人的建议）
```

### 场景C：面试准备

```
我正在准备[职位]面试，公司是[公司类型]。

背景：工作经验[x年]，技术栈：[列出]

请准备：
1. 高频面试题 15-20 道（按类别分组，每题附回答思路）
2. 项目经历包装（用 STAR 法则，基于我的经历：[简述]）
3. 反问环节：5 个高质量问题
4. 模拟面试：扮演面试官问我 10 个问题
```

## 6.5 动手练习

### 练习1：写作场景

选择一个你最常写的文档类型（周报/方案/邮件），用 6.1 节的模板设计一个可复用的提示词。

### 练习2：数据场景

设计一个"数据分析 + 报告撰写"的组合提示词（提示词链），让 AI 先分析数据再自动生成报告。

### 练习3：场景融合

为你的工作设计一个"会议纪要 → 待办事项 → 邮件通知"的提示词链。
