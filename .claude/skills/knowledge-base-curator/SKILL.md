# 知识库素材消化 SOP

## 适用范围

**仅适用于** `/Users/<your-username>/Desktop/与AI交流AI` 这个知识库。其它项目不要套用本 Skill 的规则——它们的目录结构不同。

如果用户在其它目录提到"整理笔记"，**不要**激活本 Skill。

---

## 触发后的第一动作（不要跳过）

1. 先 `Read` 项目根 `CLAUDE.md`——它定义了本库的双层结构、命名规则、决策树
2. 再 `Read` 项目根 `README.md`——它是手工维护的索引，必须保持"索引 = 实际"
3. 然后才进入下面的 SOP

## 8 条法则（落盘前 checklist，每次必走）

将这 8 条作为 TodoWrite 任务清单创建，按顺序执行：

### 法则 1：先 grep 已有，避免重复落盘
- 拿到新素材后，**先用 grep / Glob 在所有 .md 里搜核心关键词**
- 如果已有文档覆盖了 60% 以上 → **不要重复落盘**
- 改成在 README 索引里加可搜关键词、或在已有文档加章节
- ❌ 反例：图 3"Agent 五层架构"差点单独落盘，检查后发现 Harness 解剖 § 4 已覆盖

### 法则 2：小术语内嵌速查节，不单建文档
- 单个小术语（如"状态机"、"数据飞轮"）→ 内嵌到所属主文档的"§ X 术语速查"
- 用锚点链接从主表格跳转
- 等同领域术语累积到 30+ 个，再考虑单建《XX 术语手册》

### 法则 3：发现空洞 → 标记"待补全"，绝不虚标
- 引用其它文档时，如果目标文档实际没覆盖该内容
- ❌ 错误做法：硬编内容（"该文档讲了 RFT、蒸馏…"）→ 用户点过去会扑空
- ✅ 正确做法：写"该笔记侧重 X，Y 演进可与本文交叉补全"
- 后续真补全后，**回头改成正向引用**（这是法则 5 双向引用的一部分）

### 法则 4：贯穿性测试是必须的最后一步
落盘后必跑下面的 bash 脚本（见文末），检查：
- 所有跨文档目标文件存在
- 所有锚点能命中（按 GitHub Anchor 规则）
- 与已有文档的概念归属是否一致
- 内部章节顺序是否前后矛盾

### 法则 5：双向引用，不要单向链接
两篇文档互相引用时，A → B 和 B → A 都要建：
- 新文档 § "与本知识库其他章节的关联"加链接
- 同时修改被引用文档，加一条指回新文档
- 例：DeepSeek.md ↔ 模型训练技术速查.md 是 4 处反向引用

### 法则 6：README 索引必须同步
- 任何新增 / 移动 / 大改后，**立刻**改根 README
- 简介里加可搜关键词（如"五层架构图（编排层 / 记忆层 / 大模型 / 执行层 / 反馈层）"）
- 加粗重要术语让目录扫一眼能定位
- 改完后做一次 `grep` 验证 README 中的相对路径全部能命中实际文件

### 法则 7：分块落盘 + 持续汇报
- ❌ 不要"憋一个完美大版本"
- ✅ 每完成一块就汇报：章节规划 → 落盘 → 同步 README → 反向回填 → 贯穿测试 → 最终汇报
- 每步 20~30 秒内必须有可见输出（参考用户全局 CLAUDE.md 的"长任务可见输出规范"）

### 法则 8：先评分再吸收，不照单全收
对于课程 PPT / 流行图 / 公众号文章：
1. 先对内容评分（满分 10）
2. 指出说对了什么、漏掉了什么、可商榷之处
3. 再决定：完整吸收 / 部分吸收 / 不收录但加批评记录
- ❌ 反例：图 4 "Agent 架构师"只值 6.5/10（造词警告 + 漏反馈层），所以**只在 § 10 职业视角追加并附上评价**，不独立成文

## 决策树：新内容放哪？

```
新素材内容是 →
├─ 已有模块覆盖（核心概念 / 模型架构 / 提示词工程 / 应用实践 / 技术基础 / Agent工程）
│  └─ 进对应模块目录 + 更新 README 索引

├─ 元认知 / 工作流复盘
│  └─ 同上（04_探索日志，但内容是方法论而非技术）
├─ 02_提示词工程/ 内部新增
│  └─ 必须遵守它的 01_~10_ 编号体系，先看它自己的 CLAUDE.md
└─ 全新类别（不属于上述任何一项）
   └─ ⚠️ 先和用户确认是否新建模块（NN_主题/），不擅自建目录
```

## 贯穿性测试脚本（法则 4 配套）

落盘后立刻运行这套检查（按需调整文件路径）：

```bash
# 1. 验证跨文档目标文件全部存在
echo "=== 跨文档目标存在性 ==="
grep -hoE "\(\.\.?/[^)]+\.md" <新文档.md> | tr -d '()' | while read p; do
  abs="/Users/<your-username>/Desktop/与AI交流AI/$(echo $p | sed 's|^\.\./||;s|^\./||')"
  [ -f "$abs" ] && echo "✅ $p" || echo "❌ MISSING: $p"
done

# 2. 检查所有锚点能命中目标章节
# GitHub Anchor 规则：去标点 + 转小写 + 空格变 -
# 例：## 10.7.2 数据飞轮（Data Flywheel） → #1072-数据飞轮data-flywheel

# 3. 验证 README 与实际目录一致
echo "=== README 索引 vs 实际文件 ==="
grep -oE "\([^)]+\.md\)" /Users/<your-username>/Desktop/与AI交流AI/README.md | tr -d '()' | while read p; do
  abs="/Users/<your-username>/Desktop/与AI交流AI/$p"
  [ -f "$abs" ] && echo "✅ $p" || echo "❌ MISSING: $p"
done

# 4. 列出新文档章节骨架
grep -nE "^## " <新文档.md>
```

## 风格保持规则

新文档的写作风格必须与本库已有文档一致：

| 维度 | 规则 |
|------|------|
| 语言 | **中文为主**，技术术语保留英文（如 RLHF、Token） |
| 注释 | 中文注释 |
| 代码命名 | 驼峰式（如果涉及代码示例） |
| 比喻使用 | **大量使用生活类比**（厨师、自动售货机、飞轮…） |
| 视觉元素 | ASCII 图、对比表、emoji 标注（💡 ⚠️ 🌟 ✅ ❌） |
| 章节内容 | 每个核心概念都给"是什么 + 类比 + 例子 + 表格" 四件套 |
| 小白可读性 | 假设读者一窍不通，避免行话不解释 |
| **笔记脱离原图独立成立** | 用户给截图整理笔记时，**读者看不到原图**，文档不能依赖它（详见下方专项） |

### 笔记脱离原图独立成立（专项规则）

源自项目根 CLAUDE.md「基于截图/图片整理笔记的规则」。本规则只在"用户给截图/PPT/图片让你整理"的场景生效——日志、对话内已附图也属于此类。

**❌ 禁用**：所有依赖外部视觉的指代措辞

```
这张图 / 这种 PPT / 图里 / 课程图里那 4 句话
PPT 里没列出 / 配色用荧光黄 / 看图就知道
```

**✅ 改写规则**：把"这张图/PPT"换成**内容主语**——主语必须是笔记里能看到的对象

| 原话 | 改写后 |
|------|--------|
| 这张图 | 这套定义 / 这个框架 |
| 这种 PPT | 这种 title / 这类内容 |
| 图里那 4 句话 | 行业里常见的四句判断 |
| 课程图里那 4 句话的含义 | Agent 演进的 4 个深层趋势 |
| PPT 里没列出 | 该定义没提到 |

**原话里的洞察与方法论要保留，但让它独立于具体图**：

```
原话："这种 PPT 是培训课造词"      ← 依赖图
改写："这种 title 大概率是培训课造词"  ← 独立成立
```

**出处可以交代但不依赖图**：

```
✅ "出自国内某 AI 培训课"
❌ "看图就知道"
```

**自检方法（落盘前最后一步必跑）**：写完后假装读者没看过原图，从头读一遍。凡是出现 `这张图`、`这个 PPT`、`图里` 等需要外部视觉支持的句子，**立刻改写**。

## 红线（绝对不要做）

1. ❌ **不要在 02_提示词工程/ 子项目里使用根级规则**——它有自己的 CLAUDE.md
2. ❌ **不要擅自创建新模块目录**（`06_X/` 之外的新编号）——必须先问用户
3. ❌ **不要在 .claude/settings.local.json 里加东西**——除非用户明确要求
4. ❌ **不要清理或重命名 claude code skills知识/ 目录**——那是素材附件区
5. ❌ **不要把根目录搞成"散文件堆"**——除 README.md / CLAUDE.md 外，根目录不应有零散 .md
6. ❌ **不要写"虚标引用"**——宁可标"待补全"，也不要骗读者

## 完成判定（什么时候算"消化完成"）

满足以下全部条件：

- [ ] 新文档已落盘且章节骨架完整
- [ ] 所有跨文档引用的目标文件存在且锚点命中
- [ ] README 索引已同步（含可搜关键词加粗）
- [ ] 反向引用已建立（被引用文档也指回新文档）
- [ ] 贯穿性测试 0 错
- [ ] **若来源是截图/PPT**：跑过"脱离原图独立成立"自检（见风格保持规则专项）
- [ ] 给用户做了最终汇报：规模 / 章节 / 关系网 / 下一步建议

只有 7 项全打勾，才算这次消化完成。
